# 青蛙压缩 P1 核心体验改造 PRD

## 1. 背景

Pic Smaller 当前已经具备浏览器本地图片压缩能力，包括本地文件处理、Web Worker、WASM 编码、滑动对比、单张下载和 ZIP 批量打包。本轮改造不重写压缩内核，而是在低风险前提下，把产品外壳和核心流程调整为中文优先的「青蛙压缩」体验。

P1 的目标是让用户进入首页后立刻理解：这是一个中文图片压缩工具，图片不会上传到服务器，上传后会自动获得压缩结果，并且可以按需展开高级设置。

## 2. 目标

- 将默认语言切换为简体中文。
- 将核心品牌文案替换为「青蛙压缩」。
- 强化「本地压缩、隐私安全、一键压缩」的首屏认知。
- 保留现有浏览器本地压缩能力和下载能力。
- 上传后默认进入自动压缩结果页，不要求用户先配置参数。
- 新增 P1 智能压缩默认策略，并默认保留原格式。
- 默认隐藏高级设置，降低首屏干扰。
- 让压缩结果、节省比例、对比入口和打包下载更明确。

## 3. 非目标

- 不做 HEIC/HEIF 支持。
- 不做支付、会员、登录或账号体系。
- 不做 API、小程序或服务端压缩。
- 不重构压缩内核。
- 不接入 jSquash。
- 不默认转 WebP 或 AVIF。
- 不新增目标 KB 压缩、历史记录或批量格式转换。

## 4. 产品定位

产品名称：青蛙压缩

Slogan：本地压缩更安全，一键瘦身不模糊

核心隐私提示：图片不上传，本地处理，隐私不泄露

P1 应保持工具优先，首页首屏直接承载上传和压缩体验，不做营销落地页。

## 5. 用户流程

### 5.1 首次进入

用户打开首页后，应看到中文品牌、Slogan、本地隐私提示和居中的上传区域。

上传区域文案：

- 拖拽图片到这里
- 点击上传

页面应继续支持以下上传入口：

- 拖拽图片上传。
- 点击选择图片上传。
- 文件夹上传。
- 剪贴板粘贴图片。

### 5.2 上传后自动压缩

用户上传图片后，系统默认使用智能压缩策略开始压缩，不要求用户先选择输出格式、质量或其他参数。

P1 智能压缩默认保留原格式，避免自动转 WebP/AVIF 带来兼容性风险。

### 5.3 查看结果

压缩完成后，结果区应显示聚合数据：

```txt
原图大小 -> 压缩后大小 -> 节省 XX%
```

结果表格继续保留：

- 单张原图大小。
- 单张压缩后大小。
- 单张压缩率。
- 单张下载。
- 单张预览和对比入口。

单图或首图完成后，应提供更明显的「对比效果」按钮。现有 `Compare` 滑动对比能力继续复用。

### 5.4 批量下载

多图压缩完成后，用户可以继续单张下载，也可以点击「打包下载」下载 ZIP。

ZIP 文件名应改为：

```txt
frog-compress.zip
```

## 6. 功能需求

### 6.1 品牌与视觉

- 替换 `public/logo.svg` 为青蛙品牌标识。
- 替换 `Logo` 组件中的品牌呈现。
- 主题继续使用 Ant Design token。
- 主色使用薄荷绿系。
- 中文默认文案围绕「青蛙压缩」展开。

### 6.2 默认语言

- 默认语言改为 `zh-CN`。
- 本轮中文优先。
- 其他语言可以暂时保留 Pic Smaller 原文案。

### 6.3 上传区

- 上传区在首屏居中放大。
- 保留已有上传能力，不降低当前可用性。
- 上传后直接进入压缩结果流程。

### 6.4 智能压缩

新增 `smart` 默认模式，作为上传后的默认压缩策略。

P1 策略：

- JPEG：保留 JPEG 输出，使用平衡质量参数。
- PNG：保留 PNG 输出，使用现有颜色数和抖动参数。
- WebP：保留 WebP 输出，使用平衡质量参数。
- GIF：保留 GIF 输出，使用现有颜色数和抖动参数。
- SVG：保留 SVG 输出，走现有 SVG 优化路径。

### 6.5 高级设置

- 高级设置默认隐藏。
- 用户主动展开后才显示高级设置。
- PC 端不再默认占用右侧面板。
- 移动端继续使用 Drawer 展示高级设置。
- PC 和移动端共用同一语义状态。

### 6.6 结果与对比

- 压缩完成后显示聚合节省数据。
- 单图或首图完成后显示「对比效果」按钮。
- 表格预览内继续保留对比入口。
- 继续复用现有滑动对比组件。

### 6.7 下载

- 单张下载能力保持不变。
- 批量下载按钮文案改为「打包下载」。
- ZIP 文件名改为 `frog-compress.zip`。
- 现有 ZIP 重名处理逻辑保持可用。

## 7. 接口与类型变更

### 7.1 LocaleData 扩展

上传区新增字段：

```ts
slogan: string;
privacyHint: string;
```

结果区新增字段：

```ts
summaryBefore: string;
summaryAfter: string;
summarySaved: string;
compareAction: string;
downloadZip: string;
```

高级设置新增字段：

```ts
advancedSettings: string;
smartMode: string;
```

### 7.2 首页状态扩展

- 增加高级设置展开状态。
- PC 和移动端使用同一高级设置语义。
- 增加智能模式标识，默认启用。

### 7.3 新增纯函数

新增智能压缩参数解析：

```ts
resolveSmartCompressOption(fileType);
```

用途：根据文件类型返回 P1 智能压缩参数。

新增节省数据计算：

```ts
calculateSaving(originSize, outputSize);
```

用途：统一计算节省字节和百分比，供结果区和测试复用。

## 8. 验收标准

- 首屏默认显示「青蛙压缩」品牌。
- 首屏显示「本地压缩更安全，一键瘦身不模糊」。
- 首屏显示「图片不上传，本地处理，隐私不泄露」。
- 上传区居中且文案为「拖拽图片到这里 / 点击上传」。
- 拖拽、点击上传、文件夹上传、粘贴图片均可触发压缩。
- 上传单张 JPG、PNG、WebP、GIF、SVG 后能看到压缩结果。
- 压缩结果显示聚合节省数据。
- 单张下载可用。
- 单图或首图完成后能进入「对比效果」。
- 多图上传后可以单张下载，也可以打包下载。
- 打包下载文件名为 `frog-compress.zip`。
- 高级设置默认隐藏。
- 展开高级设置后可以调整参数并重新压缩。
- 移动端高级设置仍以抽屉方式展示，内容不重叠。

## 9. 测试计划

### 9.1 单元测试

- `resolveSmartCompressOption()` 覆盖 JPEG、PNG、WebP、GIF、SVG。
- `calculateSaving()` 覆盖正常压缩、压缩后变大、原始大小为 0。
- 现有 `getOutputFileName()` 测试保持通过。
- 现有 ZIP 重名逻辑测试保持通过。

### 9.2 构建与静态检查

需要运行：

```bash
npm test
npm run build
```

如果样式或组件改动较多，需要额外运行：

```bash
npm run lint
```

### 9.3 手动验收

- 检查桌面端首屏品牌、中文文案、隐私提示和上传区位置。
- 检查移动端首屏布局和高级设置 Drawer。
- 上传 JPG、PNG、WebP、GIF、SVG 各一张图片。
- 检查压缩后结果、聚合节省数据、单张下载和对比入口。
- 上传多张图片并检查 `frog-compress.zip` 下载。
- 展开高级设置，调整参数后重新压缩。

## 10. 风险与约束

- P1 不改压缩内核，因此智能压缩必须基于现有参数能力实现。
- 默认保留原格式，避免兼容性问题和用户预期偏差。
- 高级设置隐藏后，仍需保证老用户可以找到原有参数。
- GIF、SVG 的压缩路径与普通位图不同，结果展示和对比入口需要兼容现有行为。
- 当前请求优先于历史文档中「不做 UI redesign」的旧约束。
